# Product Hunt Field Specs (current — verify on the live form)

Exact submission specs for `LISTING.md` (Step 3). Specs change; these were verified against Product
Hunt's official Help Center ("How to post a product") and the official "Preparing for launch" page in
**mid-2026**. Where PH's own pages conflict, both numbers are noted — **write to the stricter one** and
confirm against the live form at submission time. Third-party numbers are labeled as such.

> Submission flow: log in → **Submit** (top right) → **New Product** → enter the product **URL first**,
> then proceed through the fields below.

## The field table

| Field | Spec (mid-2026) | Notes |
|-------|-----------------|-------|
| **Product name** | No official character limit published | Content rule (official, verbatim): *"Only the product's name, no description or emojis (unless it is a legit part of the name)."* Third-party guides cite a practical ~40 chars and the form does constrain it — keep it short. |
| **Tagline** | **60 characters max** (official, confirmed) | A very short description; PH's only style rule is *"no gimmicks or over-the-top language."* No published emoji/all-caps ban on the tagline specifically — but keep it clean and benefit-led. **Count the characters.** |
| **Description** | **500 characters max** | PH officially **extended the limit from 260 → 500** in 2025 (per PH's own 2025 recap). The older "How to post a product" help article still shows stale *"within 260 characters"* text — **500 is current.** Front-load the value (the tagline/feed do the click work); confirm on the live form. |
| **Gallery images** | **2 minimum** (gallery hidden below 2); **4–6 recommended**; **1270×760** (~1.67:1) | First image = the **launch poster** (see below). 4–6 is third-party convergence; PH official states only the 2+ minimum and the 1270×760 recommendation. |
| **Gallery file types/size** | PNG (static) / JPG / **GIF**; GIF **must be < 3MB** (official); static images: PH silent, ~5MB per third-party guidance | **GIFs do not autoplay** — they animate on hover, and the **first frame** shows statically, so design the first frame as the thumbnail. |
| **Thumbnail / logo** | **240×240** square; static image or GIF | GIF must be **< 3MB** and "not too flashy." |
| **Demo video** | **YouTube links only**; full URL (shortened links won't load); **not private** | ~30–90s recommended (third-party); design it to work **muted / with captions**. ~53% of Product-of-the-Day winners since 2021 included a video. |
| **Topics / launch tags** | **Up to 3** | Pick only the few that most strongly relate to the product; match PH's taxonomy. |
| **Shoutouts** | **3 per launch** | Credit products/tools you build on or love. |
| **Makers** | Add co-makers by their **PH username**; include yourself as a maker | Makers can join the conversation — needed so the team can reply on launch day. |
| **First / maker comment** | No published character limit | Always write one. PH's own stat: **70% of Product-of-the-Day/Week/Month winners posted a maker first comment.** See `copywriting.md` for the skeleton. |
| **Promo code / offer** | Dedicated section; **3 required fields**: "What is the offer?", "promo code", "expiration date" | Any offer must be tied to **trying / feedback**, never to upvoting (incentivized votes are banned — see `policy-compliance.md`). |
| **Interactive demo embed** | Arcade · Storylane · ScreenSpace · Hexus · Supademo · Layerpath | Optional; a live embed lifts engagement. |
| **Scheduling** | Schedule a launch up to **~1 month** in advance | Lock the date early; pick the day per `ph-mechanics.md`. |

## Pre-launch pages: what changed (important)

PH's standalone **"Coming Soon" / Teaser (Upcoming)** pages were **discontinued** (reported Aug 2025),
and the older paid **"Ship"** product was retired before that. **As of mid-2026 there is no first-party
PH teaser-page field** to build a "notify me" page on. Build pre-launch audience on **your own channels**
instead: schedule the launch, use PH **badges/embeds** on your site, and grow **your own email list** and
social following. (Older guides that tell you to build a PH "Ship"/Upcoming page are stale.)

## The first gallery image = the launch poster

The first gallery image is the highest-conversion asset: it appears in the carousel, in social shares,
and in embeds. Make it a branded **1270×760 poster** — product name + one-line value prop + the single
strongest visual — then let each subsequent slide show **one** distinct feature/workflow with a short
overlaid caption. Aim for 4–6 slides that tell a value story, not a feature dump.

## Honesty rule

When a number here is load-bearing for the user's launch (tagline length, description limit, image size)
and might be stale, **verify against the live submission form** via the `browse` skill before asserting
it. Never present a third-party number as an official PH spec — the table marks which is which.

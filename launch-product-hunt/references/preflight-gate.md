# Pre-Flight Gate (run before declaring the launch ready)

Run end to end at Step 7 and write the result into `CHECKLIST.md`. Several items are **mechanical counts —
count, don't eyeball.**

> If a single box cannot be honestly ticked, the launch is **not** ready. Fix it, then re-run.

## Positioning & grounding

- [ ] `.ulpi/launch/positioning.md` exists and every asset traces to it.
- [ ] Nothing is invented — every feature, metric, customer, and proof point is real.

## Listing assets (mechanical — count / measure)

- [ ] Product name: short; name only, no emoji unless part of the brand.
- [ ] Tagline: **≤ 60 characters** (counted) and benefit-led; one recommended option chosen.
- [ ] Description: **≤ 500 characters** (counted), value-led.
- [ ] Topics/launch tags: **≤ 3**, matched to PH's taxonomy.
- [ ] Gallery: **≥ 2 images** (4–6 recommended) at **1270×760**; first image is a self-explanatory poster
      (it's the social preview). GIF first frame reads as a static thumbnail.
- [ ] Thumbnail/logo: **240×240** (GIF < 3MB if used).
- [ ] Demo video (if any): YouTube full URL, public, ~30–90s, legible muted.
- [ ] Maker first comment: **drafted and present**, follows the story skeleton, ends on a genuine
      question, and **contains no upvote ask**.
- [ ] All external links are **UTM-tagged** for attribution.

## Compliance (zero tolerance)

- [ ] **Banned-phrase scan passes across ALL outreach and the maker comment:** zero instances of
      `upvote`, "vote for us", "support us" (vote sense), "help us hit #1", "an upvote would mean…", or any
      perk tied to a vote. (See `policy-compliance.md`.)
- [ ] Any offer is available to **everyone** and **not** conditioned on a vote.
- [ ] No plan to mass-DM identical copy to strangers or scrape cold emails.
- [ ] Maker/team accounts are **real, complete, and aged** — no new/sock accounts created to vote.

## Plan & readiness

- [ ] Launch **date/day chosen** (Tue–Thu default; weekend only as a deliberate low-competition play) and
      **scheduled for 12:01am PT**.
- [ ] Hunter decision recorded (self-hunt by default; **no paid hunt** — that risks removal/ban).
- [ ] Supporter list is **realistic for the target** (a warm audience exists; segments + send waves
      planned). If reach is thin, that's stated honestly, not papered over.
- [ ] Launch-day runbook is written and (if a team) roles are assigned.
- [ ] Measurement is wired: UTM scheme set and signup/activation tracking ready (`launch-analytics`).
- [ ] Post-launch plan exists (badge, convert-the-spike, thank-yous).

## Expectations (honesty check)

- [ ] The user has a **realistic outcome range** and understands featuring/Product of the Day is **not
      guaranteed**, and that PH is a visibility/credibility amplifier — not a primary acquisition channel.

## Scored self-critique (calibrate — don't rubber-stamp)

Score the launch package 0–4 on each axis. Be honest — a **4 means genuinely excellent**; most real work
lands 2–3. Axes: **positioning clarity · tagline strength · gallery story · first-comment quality ·
outreach compliance · plan/runbook completeness · expectation realism**.

- [ ] Total recorded (max 28). Any axis **≤ 2 → name the specific fix, apply it, and re-score.**
- [ ] Revise-and-justify: for every change made at this gate, state in one line WHAT changed and WHY. Do
      not declare the launch ready with an un-actioned axis ≤ 2 or any compliance box unticked.

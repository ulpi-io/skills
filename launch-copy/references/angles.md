# Angle Framework (A/B/C/D)

Draft every headline/tagline/one-liner across **distinct angles**, not as reworded twins. Four proven
angles — generate options across them, then recommend the one that reads best cold for this product and
channel. (Borrowed from the way strong directory/launch copy is versioned to avoid duplicate-content and
to find the framing that lands.)

| Angle | What it leads with | Best when |
|-------|--------------------|-----------|
| **A — Feature-led** | the standout capability | the capability itself is novel/striking |
| **B — Benefit-led** | the outcome / pain removed | almost always the default winner |
| **C — Comparison / alternative** | a known product you're better/cheaper/simpler than | the reference product is widely known |
| **D — Audience-led** | who it's for, named | a sharp, specific ICP ("for indie iOS devs") |

## Tagline / headline patterns (≤ ~60 chars where the channel caps it)

Core formula (outcome-first, implementation-last): `[action verb] + [outcome] + [for whom OR without the pain]`

| Angle | Pattern | Example |
|-------|---------|---------|
| B | "[outcome] in [time]" | "Turn meeting notes into action items in 2 minutes" |
| B | "[outcome] without [pain]" | "Ship client reports without manual spreadsheets" |
| A | "[capability] for [who]" | "Design faster with a reusable component library" |
| C | "[BigProduct] alternative" / "[A] + [B] for [who]" | "Open-source Calendly alternative" · "Gumroad + Linktree alternative for creators" |
| D | "Built by [who] for [who]" / "Never [pain] again, built by [who]" | "Never be late again, built by a team with ADHD" |

Real strong taglines to calibrate against: *"Turn boring product images into beautiful photos with AI"*
(Pebblely) · *"Cut down meetings by 40% — interactive async video for work"* (Tape) · *"Ditch the grids,
create websites like you design graphics"* (WebWave) · PH's own model line *"Send your friends a voicemail
meow from a real cat"* (concrete, specific, zero hype).

## Description tiers

When a channel wants both, derive both from the same positioning:

- **Short (≈ one line / 150–300 chars):** `[Product] is [category] that helps [who] [outcome].`
- **Long (≈ 400–500 chars):** short + one or two concrete capabilities in plain language + one
  differentiator or a free/open-source hook. No fluff.

## Launch post / first comment

This is a story, not a feature list — the structure varies by channel (a PH maker comment vs a Show HN
first comment differ in tone), so fill the **skeleton the asset profile provides** in the profile's voice.
Universal rules: open with the origin/problem, be specific about what it does and what it doesn't yet,
ask a **genuine question** at the end, and obey the channel's compliance rule (e.g. never solicit upvotes
on PH). See `asset-profiles.md` for the per-channel skeletons.

## Picking the recommendation

Recommend the version that (1) reads clearest cold, (2) is most specific/true to the product, (3) fits the
channel's voice, and (4) names the audience when that adds clarity. State the reason in one line.
